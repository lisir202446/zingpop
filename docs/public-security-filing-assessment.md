### 用户真实身份核验及注册信息留存措施

如果生产环境手机号短信注册还没有完整验证，使用这个受控阶段表述：

```text
网站目前处于内测阶段，用户由运营管理人员人工登记和审核后开通账号；正式开放注册后，采用手机号短信验证码、账号密码等方式进行用户身份核验。系统留存用户账号、手机号、注册时间、登录时间、登录IP、浏览器/设备信息、账号状态等注册和登录信息，并由安全管理负责人定期检查，相关记录按规定保存。
```

如果生产环境手机号短信注册已经完整验证，使用这个更强表述：

```text
网站采用手机号短信验证码和账号密码方式进行用户身份核验。系统留存用户账号、手机号、注册时间、登录时间、登录IP、浏览器/设备信息、账号状态等注册和登录信息，并由安全管理负责人定期检查，相关记录按规定保存。
```

### 日志信息及用户发布信息记录留存措施

```text
网站通过服务器、Nginx、应用服务和数据库记录用户登录、访问、项目操作、请求时间、来源IP、访问URL、操作类型、浏览器/设备信息、异常错误等日志。日志文件设置访问权限，仅限授权管理人员查看；重要日志定期备份，按规定保存不少于六个月。发现异常访问、攻击行为或违法违规内容时，保留相关记录并及时处置。
```

工程备注：当前基础设施已有服务器和 Nginx 基础日志，但公开商业化上线前仍需补齐应用级审计日志，包括登录、项目访问、命令执行、模型调用、凭证变更和管理员操作。

### 违法有害信息防范处置和记录保存措施

```text
网站不提供公开通讯群组、论坛、评论区等社交发布功能。对用户账号名称、昵称、项目名称、输入内容、生成内容等信息，运营人员定期巡查；发现违法违规、有害信息或用户投诉举报后，及时采取删除、屏蔽、限制账号、暂停服务等处置措施，并保存处置时间、处置人员、处置原因和相关证据记录。
```

### 个人信息保护及风险控制技术措施

```text
网站坚持最小必要原则收集用户信息，主要收集账号、手机号、登录和安全日志等运营所需信息；网站全站使用HTTPS加密传输，服务器和数据库账号采用权限控制，生产密钥不进入代码仓库，后台访问限于授权人员。对用户数据进行备份和访问控制，支持用户提出账号和数据删除申请。网站不提供公开群组、论坛、评论等社会动员功能；对异常注册、异常访问、违法违规内容和滥用行为采取人工巡查、账号限制、内容删除、日志留存等措施。
```

工程备注：大范围商业化上线前，需要发布隐私政策、数据处理说明、账号注销 / 数据删除流程、第三方处理方披露和模型供应商披露。

## 投诉举报机制

| 字段 | 建议填写 |
| --- | --- |
| 是否建立投诉举报机制 | `是` |
| 投诉举报机制的途径 | 优先选择 `电子邮箱`；如果可以多选，再选择 `电话`；如果只有 `其他`，填写 `电子邮箱、电话` |

备案提交前或提交后，应尽快在网站页脚或联系我们页面公开真实可用的投诉邮箱和联系电话。

### 投诉举报机制的描述

```text
网站已建立投诉举报处理机制。用户可通过网站公布的联系邮箱、联系电话等方式提交违法违规信息、侵权内容、账号异常、数据安全等投诉举报。安全管理负责人收到举报后进行登记、核查和处置；对确认存在违法违规或有害信息的内容，及时采取删除、屏蔽、限制账号、暂停服务等措施，并保存举报时间、举报内容、处置结果和相关证据记录。
```

### 为监管部门和执法部门提供技术、数据支持和协助的工作机制

```text
网站指定安全管理负责人负责对接监管部门和执法部门。收到依法提出的协查、取证、处置或整改要求后，及时核验要求来源并配合提供必要的账号信息、访问日志、操作记录、IP地址、时间戳、内容记录等数据支持；对违法违规信息及时采取删除、屏蔽、限制访问、暂停账号等处置措施，并保存协助处理和整改记录。
```

## 当前能力评估

| 领域 | 当前状态 | 备案表述风险 | 商业化前动作 |
| --- | --- | --- | --- |
| HTTPS / 域名 / 服务器链路 | 基本完成 | 低 | 只保留公网 `80/443`，公网关闭 `3000/3001/4096`。 |
| 手机号认证和短信 | 尚未完整生产验证 | 中 | 验证生产数据库、迁移、短信签名 / 模板 / AK、注册、登录和密码重置。 |
| 工作台登录链路 | 需要登录态生产验证 | 中 | 验证 `www.zingpop.cn` 与 `app.zingpop.cn` 的 Cookie、WebSocket、API、文件和模型请求。 |
| 多用户隔离 | 最大阻塞项 | 高 | 面向非受信用户开放前，实现账号 / 工作区 / 项目 / 文件系统隔离和跨用户拒绝访问测试。 |
| 日志和审计记录 | 有基础服务器日志，但应用审计不完整 | 中 | 增加登录、项目访问、命令执行、模型调用、凭证变更和管理员操作审计日志。 |
| 投诉举报 | 可以先建立机制，但必须公开联系方式 | 中 | 发布投诉邮箱 / 电话，并保留内部处理登记表。 |
| 隐私和数据权利 | 文档和流程不完整 | 高 | 发布用户协议、隐私政策、数据处理说明、账号注销和数据删除流程。 |
| AI / 模型供应商合规 | 公开 AI 服务前需要复核 | 中 | 复核模型供应商商业条款、数据保留政策、第三方披露和备案 / 安全评估适用性。 |
| 支付 / 商业闭环 | 未达到生产可用 | 高 | 完成真实微信支付 / 支付宝商户凭证、回调、订单状态、充值、订阅、退款、发票和对账。 |
| 可靠性 / 恢复 | 未达到付费用户标准 | 中 | 增加备份、恢复演练、日志轮转、监控告警、事故预案和重启恢复检查。 |
